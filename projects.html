Capstone Project

Open-Source Security Scanner

View PDF

This capstone project focused on building an open-source security scanning solution for Northeastern ITS. The goal was to reduce reliance on proprietary scanning platforms and provide a custom, scalable alternative that could fit the university's operational requirements.

The solution was designed around a Jenkins-driven pipeline that accepts a text or CSV list of IP addresses, creates build folders, and iteratively runs selected security tools against each target. Scan output is generated in structured formats and consolidated through Python processing for reporting.

The project selected OWASP ZAP for web application scanning, Nikto for web server and application checks, and testssl.sh for SSL/TLS service-level assessment. These tools were chosen because they are open source, actively maintained, and suitable for repeatable security assessments.

The final architecture emphasized scalability, simple reporting, privacy, and extensibility. Future work included integration with Microsoft Sentinel, running the solution beyond sandbox testing, tuning latency, and potentially releasing the project for broader community use.

Network Security Research

Covert Network Channel

View PDF

This project investigated covert network channels as a post-exploitation technique for stealthy data movement. The work explored how attackers can hide communication inside legitimate network behavior to avoid detection by administrators and defensive monitoring systems.

The implementation used TCP source port numbers as the covert carrier. Data was converted, encrypted, hashed, and transmitted in small chunks through crafted TCP packets between a client and server, with the server reconstructing and decrypting the hidden message.

The project evaluated the channel using Wireshark packet captures and Snort intrusion detection. The team compared the covert traffic against normal traffic and verified that encrypted transmission did not reveal clear payload content in packet analysis.

The results showed successful end-to-end communication while avoiding Snort alerts in the tested setup. The report also identified future improvements, including adaptive protocol selection, timing changes based on baseline traffic, and alternative carriers such as DNS, HTTP, or HTTPS.

Wireless Security Research

Deauthentication Attack in IEEE 802.11 Networks

View PDF

This research paper examined deauthentication attacks against IEEE 802.11 wireless networks. The project focused on the weakness of management frames, which are not authenticated or encrypted in the same way as protected data traffic.

The paper explained how attackers can spoof deauthentication or disassociation frames and cause clients to disconnect from a legitimate access point. It framed the attack as a MAC-layer denial-of-service issue that can be launched with inexpensive, readily available hardware.

The experimental analysis used an ESP8266-based NodeMCU deauther module with an OLED display. The setup scanned nearby access points, selected a target, and triggered deauthentication, probe, or beacon-frame attacks through a local web interface.

The research also reviewed defensive ideas such as pseudo-random sequence number validation and authentication-based frame verification. The project highlighted how wireless management frame weaknesses can create practical availability risks even when higher-level wireless encryption is in place.
